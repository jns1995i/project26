Sizing WIDTH · HEIGHT · AREA

The sizing system provides three naming conventions: pixel-based (exact px values), percentage-based (relative to parent), and the shorthand area (equal width & height). Mix and match to suit your layout needs.

Quick Reference — Naming Conventions

Pixel classes: prefix + px + number → wpx200, hpx100, top50
Percentage classes: prefix + number → width50, height100, t50
Min/Max: prefix + Min/Max + number → widthMin30, hpxMax400
Colors: category + ColorName + shade → textBlue500, bgRed200, borderGreen300
Equal square: area{n} → sets both width and height to npx
Negative margins: same as positive with a n suffix or negative prefix convention
Font weight shortcuts: str1–str9 or named (bold, semiBold, etc.)
